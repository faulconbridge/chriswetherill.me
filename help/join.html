SQL JOIN Statement

SQL joins are used to query data from two or more tables, based on a relationship between certain columns in these tables.

The JOIN keyword is used in an SQL statement to query data from two or more tables, based on a relationship between certain columns in these tables.

Tables in a database are often related to each other with keys.

A primary key is a column (or a combination of columns) with a unique value for each row. Each primary key value must be unique within the table. The purpose is to bind data together, across tables, without repeating all of the data in every table.

Before we continue with examples, we will list the types of JOIN you can use, and the differences between them.

  • JOIN: Return rows when there is at least one match in both tables
  • LEFT JOIN: Return all rows from the left table, even if there are no matches in the right table
  • RIGHT JOIN: Return all rows from the right table, even if there are no matches in the left table
  • FULL JOIN: Return rows when there is a match in one of the tables

SQL JOIN Syntax

SELECT column_name(s)
FROM table1
RIGHT JOIN table2 ON table1.ColName1 = table2.Colname2

Example

Okay, so this is actually a kind of nontrivial example and is overkill for a simple example. But whatevs. Let's say we want a word list that:

  • Includes the cue, cue log frequency, cuePOS, target, target log frequency, number of P responses for each cue, and the number of P responses for each target;
  • Only returns values with measured log frequencies;
  • Only returns homographs;
  • Only returns cues whose log frequency is 1.5 < x < 2.8;
  • Only returns cues that can be BOTH nouns and verbs;
  • Is returned in alphabetical order by target word

That's a whole lot of craziness, but you still following? So, basically, these data that we want reside in two separate tables: ELP has some characteristics (like log freq) and USF has others (like targets and a homograph marker). However, we can functionally combine these tables with a JOIN statement. Here we use two: one joing ELP.WORD and USF.CUE; the other joining ELP.WORD and USF.TARGET. But wait, how can we do that? We can't combine three variables like that, can we? Well, we kind of can. Basically, with the aliases and some other magic we're keeping the two joins independent of one another. So think of it as though we took ELP and cloned it. So now we have ELP1 and ELP2. We join ELP1 along ELP.WORD = USF.CUE and we join ELP2 along ELP.WORD = USF.TARGET. Pretty darned close to functionally independent of one another, right?


SELECT e1.WORD as Cue, e1.LgSUBTLWF as word_frq,
e1.POS, u1.TARGET, e2.LgSUBTLWF as tgt_freq, u1.SAMPLE, u1.N
FROM USF u1
RIGHT JOIN ELP e1 ON (u1.CUE = e1.WORD)
RIGHT JOIN ELP e2 ON (u1.target = e2.word)
WHERE e1.LgSUBTLWF is NOT NULL AND u1.CueHomograph is NOT NULL
AND (e1.POS='NN|VB' OR e1.POS='VB|NN') AND e1.LgSUBTLWF>1.5
AND e1.LgSUBTLWF>2.8 AND e1.LENGTH>3 AND e1.LENGTH<7
GROUP BY u1.TARGET
ORDER BY e1.WORD ASC
